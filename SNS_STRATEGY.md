## 1. ファクトチェック結果

### 1.1 X (Twitter) API料金体系

| ティア | 月額 | 書込 | 読取 | 判定 |
|--------|------|------|------|------|
| Free | $0 | 500件/月（≈16/日） | 不可 | ✅ 正確 |
| Basic | $200 | 50,000件/月 | 15,000件/月 | ✅ 正確 |
| Pro | $5,000 | 300,000件/月 | 1,000,000件/月 | ✅ 正確 |
| Enterprise | $42,000+ | カスタム | カスタム | ✅ 正確 |

**2026年2月〜 従量課金制（Pay-As-You-Go）追加**: 正確。Basicの$200固定を回避する
中間オプションとして提供されたが、月500〜2000投稿規模なら結局$100〜200/月に。

### 1.2 アダルトアフィリエイト直リンク投稿のリスク

- **アダルトコンテンツ投稿自体**: X公式では許可（センシティブ設定が必要）✅
- **有料パートナーシップラベル義務**: 商業プロモーションに必須 ✅
- **アダルト産業はラベル禁止カテゴリー**: ✅ 確認済み
- **→ デッドロック論**: 概ね正確。ただし「オーガニック投稿として個人的感想風に書く」
  抜け道は実務上存在するが、大量自動投稿では即シャドウバン対象になる

### 1.3 Bluesky APIの制限

- **5,000件/時 無料**: ⚠️ 要注意。AT Protocolのレートリミットは
  厳密には「300req/5min」等の細かい制限があり一律5,000/hではないが、
  Xの月500件と比較して圧倒的に寛大なのは事実 ✅
- **アダルトコンテンツ許可（要ラベル）**: ✅ 確認済み
- **オープンAPIで自動化しやすい**: ✅ `@atproto/api` パッケージで実装可能

### 1.4 Telegram Bot API

- **無料・制限なし**: ✅ ボット経由の投稿はレートリミットが非常に緩い
- **プライベートチャンネルで直リンク配信可能**: ✅
- **アダルトコンテンツの検閲**: Telegramは公式に規制しているが、
  プライベートチャンネルは実質的にほぼ無検閲 ✅

### 1.5 既存コードの問題点

```
x_autopost.js の致命的な問題:
1. sql.js（ローカルSQLite）を使用 → Turso移行済みなので動かない
2. Twitter APIクライアントは未実装（TODO状態）
3. アフィリエイト直リンクを直接投稿するアーキテクチャ → ToS違反
4. x_posted_at カラムをMGS DBに追加していない（schema未更新）
5. desireav-001 が未定義（002〜008のみ）
```

## 2. 戦略の再設計

### 2.1 新アーキテクチャ（ファネル構造）

```
[X / Bluesky] ──クッションページURL──▶ [lunar-zodiac.vercel.app]
                                              │
                                              ▼
                                    作品詳細ページ（OGP最適化）
                                    非露骨サムネ + テキストレビュー
                                              │
                                    ┌─────────┴─────────┐
                                    ▼                   ▼
                              FANZA購入リンク      MGS購入リンク
                              （アフィリエイト）   （アフィリエイト）

[Telegram Bot] ──────────────────────────────▶ 直リンク配信（登録者限定）
```

**核心原則**: XとBlueskyは「集客口（トップファネル）」に徹する。
直リンクはTelegramのみ。

### 2.2 プラットフォーム別戦略

#### X（旧Twitter）
- **投稿内容**: サイトURL + テキストレビュー（感想風）のみ
- **API戦略**: 月500件（Free）で十分 → 1日1〜2投稿 × 複数アカウント
- **投稿アーキテクチャ**:
  - 親ツイート: キャッチコピー + サムネOGP（非露骨）+ **lunar-zodiacのURL**
  - 直アフィリエイトリンクは**完全に排除**
- **必要コスト**: $0（Free tier）で運用可能

#### Bluesky（メインSNS基盤）
- **投稿内容**: Xと同様 + アダルトラベル付きコンテンツ
- **API**: `@atproto/api` npm パッケージで無料実装
- **優位性**: アルゴリズムが比較的透明、アダルト許可、API無料
- **実装**: n8n不要 → 既存node.jsスクリプト基盤で直接実装

#### Telegram
- **運用形態**: プライベートチャンネル（ジャンル別）
- **Bot機能**: 新作自動通知 + キーワード検索 → 作品推薦
- **直リンク配信**: FANZA/MGSアフィリエイトリンクをここでのみ解禁
- **実装**: `node-telegram-bot-api` または `telegraf` npm パッケージ

### 2.3 コスト試算

| 項目 | 旧戦略 | 新戦略 |
|------|--------|--------|
| X API | $200/月（Basic想定） | $0（Free tier）|
| Bluesky API | - | $0 |
| Telegram Bot API | - | $0 |
| n8n VPS | 不要 | 不要（既存node.jsで対応）|
| **合計** | **$200/月** | **$0/月** |

## 4. シャドウバン対策チェックリスト

### X投稿ルール
- [ ] 1アカウント1日16件以内（Free tier上限以内で収める）
- [ ] URLなしのテキストのみ投稿を20〜30%混入
- [ ] 投稿間隔にランダムジッター（±15〜45分）
- [ ] 複数アカウント間で**同一テキスト禁止**（LLM rewrite必須）
- [ ] アフィリエイトリンク直投稿**完全禁止**
- [ ] サムネはOGP経由（センシティブフラグ不要なら設定しない）

### アカウント管理
- [ ] アカウントごとに異なるIPアドレス（住宅用プロキシ推奨）
- [ ] ブラウザフィンガープリント分離（MultiloginやIncogniton）
- [ ] アカウントのウォームアップ期間（新規作成後2週間は手動投稿）
